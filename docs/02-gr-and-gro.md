# .gr and .gro

## .gr

`.gr` is source that stays closest to the root.

Example roles:

- raw byte source
- GrASM source
- GrLang source
- kernel source
- driver source

## .gro

`.gro` is a build product.

Example roles:

- boot image
- kernel image
- executable app
- library package
- SDK bundle

## Meaning

Public meaning:

```txt
.gr  = grain / ground / root
.gro = grown form
```

The original private meaning stays implicit.
